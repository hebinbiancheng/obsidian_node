# AC 加贝叶斯的内容识别工程流程

## 1. 为什么是 AC + 贝叶斯，而不是二选一

原文的核心判断是：

```text
AC + 贝叶斯 > max(AC, 贝叶斯)
```

含义是：

- 只用 AC：召回快，但误伤高；
- 只用贝叶斯：面对长尾社区语言和低频表达，效果不稳定；
- 组合使用：AC 先圈定类别，贝叶斯在类别内精确过滤。

## 2. 识别流程

```text
用户内容
  -> 文本清洗与标准化
  -> AC 自动机匹配主关键词
  -> 根据主关键词圈定垃圾类别
  -> 读取该类别策略
  -> 结合可选词、共现词、贝叶斯概率
  -> 判断垃圾概率或命中策略
  -> 输出处理动作
```

## 3. 主关键词

主关键词用于确定候选类别。例如：

- 赌博类主关键词；
- 色情类主关键词；
- 广告类主关键词；
- 攻击辱骂类主关键词；
- 灰产引流类主关键词。

主关键词的作用不是最终判定，而是让后续策略只在相关类别中运行。

## 4. 可有可无词

原文图三中的“可有可无”可以理解为辅助词、上下文词、共现词。

这些词可能单独出现时不构成违规，但与主关键词组合时能显著提高垃圾概率。例如：

```text
主关键词 + 联系方式
主关键词 + 地域词
主关键词 + 价格词
主关键词 + 引导动作
```

## 5. 策略生成

策略生成通常来自样本分析：

1. 收集垃圾样本和正常样本；
2. 分词或提取候选 token；
3. 统计每个词在垃圾/正常样本中的出现频率；
4. 用贝叶斯公式计算区分度；
5. 结合人工经验生成策略；
6. 离线验证准确率和召回率；
7. 达标后上线。

## 6. 系统架构抽象

```text
样本库
  ├─ 垃圾样本
  └─ 正常样本
      ↓
特征统计 / 贝叶斯分析
      ↓
策略配置平台
      ↓
离线验证
      ↓
线上 AC + 策略过滤服务
      ↓
识别结果 / 运营反馈 / 新样本沉淀
```

## 7. 处理动作

内容识别结果可以对应不同动作：

- 直接拦截；
- 进入人工审核；
- 降权或折叠；
- 风险打标；
- 加入样本池用于后续训练；
- 对高风险账号触发更严格风控。
