# AC 自动机核心概念与多模式匹配

## 1. AC 自动机解决什么问题

AC 自动机，即 Aho-Corasick Automaton，用于在一段文本中同时匹配多个模式串。它适合：

- 敏感词匹配；
- 黑名单词库扫描；
- 多关键词召回；
- 内容安全初筛；
- 日志/文本流中的模式识别。

如果有几万、几十万个关键词，逐个关键词扫描文本会很慢。AC 自动机通过 Trie + fail 指针，将多模式匹配压缩为一次线性扫描。

## 2. 核心结构

AC 自动机通常包含三部分：

1. **Trie 树**：保存所有模式串；
2. **fail 指针**：当当前路径匹配失败时，跳到最长可匹配后缀状态；
3. **输出集合**：某些节点标记一个或多个模式串匹配成功。

## 3. 时间复杂度

构建完成后，对长度为 `n` 的文本进行匹配，复杂度接近：

```text
O(n + 命中结果数量)
```

这使它非常适合在线内容扫描。

## 4. 示例

原文使用模式串：

```text
he / she / his / hers
```

构建 AC 自动机后，输入：

```text
ushers
```

会匹配到：

- `she`
- `he`
- `hers`

## 5. 在垃圾内容识别中的作用

在该系统里，AC 自动机不是直接给最终判定，而是承担“召回”和“类别圈定”：

```text
内容文本
  -> AC 匹配主关键词
  -> 判断可能属于 色情 / 赌博 / 广告 / 攻击 等类别
  -> 进入该类别对应的精细过滤策略
```

这样设计的好处是：

- 速度快；
- 适合多类别关键词库；
- 可解释性强；
- 便于运营配置；
- 为后续贝叶斯策略缩小判断范围。

## 6. Double Array Trie

原文提到可以用 Double Array Trie 实现 AC 自动机。它的意义在于：

- 节省 Trie 存储空间；
- 提升数组访问效率；
- 适合大规模关键词词典；
- 常用于工业级分词、敏感词过滤和文本匹配。
