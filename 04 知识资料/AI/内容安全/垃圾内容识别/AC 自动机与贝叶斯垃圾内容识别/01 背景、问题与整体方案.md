# 背景、问题与整体方案

## 1. 业务背景

开放社区会产生大量 UGC 内容，包括评论、私信、回答、提问等。高质量讨论依赖友善、理性、专业的内容环境，但垃圾制造者也会利用开放入口发布：

- 违法内容；
- 色情、赌博、灰产广告；
- 微商、代开发票等垃圾广告；
- 辱骂、人身攻击、不友善内容；
- 诱导、欺诈、引流信息。

这些内容会破坏用户体验，也会增加社区治理成本。

## 2. 旧方案的问题

原文提到此前尝试过文本分类模型、DFA 关键词召回、人工审核等方式，但都存在短板。

| 方案 | 优点 | 问题 |
|---|---|---|
| 文本分类模型 | 能学习统计特征 | 召回不足，长尾词和新套路适应慢 |
| DFA / 关键词 | 召回快，工程简单 | 误伤高，语义和上下文理解弱 |
| 人工审核 | 判断准确，能处理复杂case | 延迟高，人力成本高，容易堆积 |

## 3. 垃圾内容的工程特征

文章的关键观察是：垃圾内容虽然形式多变，但往往“有套路”。例如：

- 固定主关键词；
- 固定诱导词；
- 某些词组经常共现；
- 某些类别下有独特表达模板；
- 攻击、广告、色情、赌博各自有明显词簇。

这意味着可以先用规则和多模式匹配快速圈定候选，再使用统计方法降低误伤。

## 4. 整体方案

```text
输入内容
  -> 文本预处理
  -> AC 自动机多模式匹配
  -> 命中主关键词，圈定垃圾类别
  -> 类别内使用贝叶斯思想计算词语区分度
  -> 结合可选词、共现关系、空间结构形成策略
  -> 输出识别结果 / 进入审核 / 拦截处理
```

## 5. 方案定位

这个系统不是一个纯机器学习模型，也不是简单敏感词过滤，而是一个**规则召回 + 统计过滤 + 运营策略闭环**的内容安全系统。

其工程价值在于：

- AC 自动机保证大词典下的高效匹配；
- 贝叶斯方法帮助筛出高区分度词，降低误伤；
- 策略管理平台让运营可以自助迭代；
- 离线验证保障策略上线前可评估准确率和召回率。
