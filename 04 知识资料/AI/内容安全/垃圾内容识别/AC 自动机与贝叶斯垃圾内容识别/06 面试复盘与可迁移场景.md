# 面试复盘与可迁移场景

## 1. 面试中如何讲这个系统

可以按“问题 -> 旧方案缺陷 -> 组合方案 -> 工程闭环 -> 指标”来讲。

### 30 秒回答

为了识别社区垃圾内容，我们不能只靠关键词，也不能只靠分类模型。关键词召回快但误伤高，纯模型面对长尾和新套路容易漏召。因此可以用 AC 自动机做多模式匹配，先根据主关键词圈定候选类别；再用贝叶斯思想统计词在垃圾样本和正常样本中的区分度，形成类别内策略；最后通过离线验证和运营自助平台持续迭代策略，以平衡准确率和召回率。

## 2. 面试官可能追问

### Q1：为什么不用普通 Trie？

普通 Trie 可以匹配前缀，但多个模式串扫描时需要频繁回退。AC 自动机通过 fail 指针复用最长后缀匹配状态，可以在线性时间内完成多模式匹配。

### Q2：AC 自动机为什么容易误伤？

因为它只判断字符串是否命中词典，不理解上下文。某些词在垃圾内容和正常内容中都可能出现，直接拦截会伤害正常用户。

### Q3：贝叶斯方法解决了什么？

贝叶斯方法通过样本统计衡量某个词出现后内容属于垃圾类别的概率，帮助筛出高区分度词，也可以识别正常倾向词，从而降低误伤。

### Q4：如何评估效果？

核心指标是准确率、召回率、误伤率、漏放率、人工审核节省量和处理延迟。上线前要离线验证，上线后要持续回流样本。

### Q5：如何处理新型垃圾内容？

通过线上样本回流、人工标注、策略挖掘、自动归纳和灰度上线机制，快速更新词库和策略。

## 3. 可迁移场景

这套方法不仅适合垃圾评论，还可迁移到：

- 舆情识别；
- 广告与营销内容过滤；
- 私信骚扰识别；
- 违规问答识别；
- 内容分类；
- 客服工单分流；
- 日志关键事件识别；
- 安全告警规则召回。

## 4. 与现代方案的结合

在现代系统中，可以把它作为第一层高效过滤器：

```text
规则/AC 快速召回
  -> 轻量分类模型
  -> 大模型复杂语义审核
  -> 人工复核
  -> 样本回流与策略更新
```

这种分层结构的优势是：

- 低成本处理大量简单 case；
- 复杂 case 交给模型或人工；
- 策略命中可解释；
- 能快速响应新垃圾套路。
