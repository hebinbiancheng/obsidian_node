# Agent Prompt Skill 面试题库索引

## 学习路径

1. 先读 [[01 Agent 面试核心问题]]，建立 Agent 与 Workflow、RAG、工具调用、多 Agent、生产化治理的答题框架。
2. 再读 [[02 Prompt 工程与 Skill 面试题]]，补齐 Prompt 设计、Few-shot、CoT、结构化输出和 Skill 使用的基础问答。
3. 最后读 [[03 Skill Token 管理与工程化]]，把 Skill 设计从“会写”提升到“能控制上下文成本、可维护、可复用”。
4. [[04 原文解析要点与来源索引]] 保留 3 篇原文的来源、结构和可追溯信息。

## 核心地图

- Agent 设计：LLM + 工具 + 记忆 + 规划，生产系统通常采用 Workflow 与 Agent 混合架构。
- RAG 与工具调用：面试重点不只是概念，而是召回质量、工具幂等、防死循环、观测和成本控制。
- Prompt 工程：清晰任务、上下文、输出格式、示例、测试集评估是基本闭环。
- Skill 工程：元数据、SKILL.md、references/scripts/assets 的分层加载，是控制 Token 和复杂度的关键。
