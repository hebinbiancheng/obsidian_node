# CDN 缓存面试题：原理 + 对比 + 深挖

> 原文：[微信文章](https://mp.weixin.qq.com/s/kObBshUEQGQFygHwn8j3Jg) · 2026-05-26 · 京东二面
> 原始资料：`^[raw/articles/wechat-cdn-cache-2026.html]`

---

## 一句话总结

CDN 本质是**反向代理缓存服务器**，部署在全球边缘节点，配合 DNS 智能解析（GSLB），让用户就近获取缓存数据——源站 QPS 能降低 90% 以上。

---

## 一、CDN 工作流程（三层架构）

```
用户层（不同地域）
  ↓ DNS 智能解析（GSLB）
边缘节点层（就近缓存命中/回源）
  ↓ 未命中才回源
源站层（业务服务器）
```

| 场景 | 延迟 | 源站压力 |
|------|------|---------|
| 无 CDN | 高（跨地域） | 大（所有用户直连） |
| 有 CDN | 毫秒级（就近） | 低（90%+ 请求被拦截） |

---

## 二、核心机制

### 1. DNS 智能解析（GSLB）

北京用户和上海用户访问同一域名 → CDN 的 GSLB 根据用户 IP + 节点负载 → 返回不同边缘节点 IP。

### 2. 缓存命中与回源

```
接收请求 → 查本地缓存 → 命中（X-Cache: HIT）直接返回
                         → 未命中（MISS）→ 向源站拉取 → 缓存 → 返回
```

响应头 `X-Cache: HIT/MISS` 判断命中状态。

### 3. 缓存过期与刷新

| 机制 | 说明 |
|------|------|
| `Cache-Control: max-age=3600` | 3600 秒内直接用缓存 |
| `Expires` | 绝对过期时间（HTTP/1.0，优先级低） |
| `ETag / Last-Modified` | 过期后条件请求，未变返回 304 |

**更新不及时的解法**：静态资源文件名加 hash 指纹（`app.a3b2c1.js`）→ 每次更新文件名变了，自然避开旧缓存。推荐 + 主动刷新 API + 缩短 max-age（需权衡）。

### 4. 防盗链

Referer 鉴权、IP 黑白名单、**URL 签名**（生成带过期时间的加密 URL）——生产环境视频网站最常用。

---

## 三、CDN 适合缓存什么

| ✅ 适合 | ❌ 不适合 |
|---------|----------|
| 静态资源（JS/CSS/图片/字体） | 实时接口（股票行情） |
| 大文件下载（安装包/视频） | 用户个性化数据 |
| 直播/点播视频流 | 频繁变更的动态数据 |
| 不常变更的 API（商品详情） | 交易请求 |

> 读多写少、更新频率低、实时性要求不高 → CDN。

---

## 四、三层缓存对比

| 维度 | 浏览器缓存 | CDN 缓存 | Nginx 缓存 |
|------|-----------|---------|-----------|
| **位置** | 用户本地 | 边缘节点 | 源站机房 |
| **服务范围** | 仅当前用户 | 同区域所有用户 | 经过该 Nginx 的用户 |
| **延迟** | 最低（本地） | 低（就近） | 中（源站机房） |
| **命中率** | 取决于单用户 | 高（大量用户共享） | 较高 |
| **源站保护** | 无 | 强（拦截大部分） | 中等 |

> 浏览器缓存 = 自家冰箱存货，CDN = 小区门口便利店

完整链路：`用户 → 浏览器缓存 → CDN 边缘 → Nginx → 源站`，越往后请求越少。

---

## 五、CDN vs Redis 缓存

| 维度 | CDN | Redis |
|------|-----|-------|
| **缓存位置** | 边缘节点（用户侧） | 服务器内存 |
| **缓存内容** | 静态资源、大文件 | 热点数据、Session、计算结果 |
| **面向用户** | 终端用户就近 | 服务内部加速 |
| **一致性** | 弱（分钟级过期） | 强（秒级/实时） |

---

## 面试答题模板

> CDN 核心是「就近缓存 + 智能调度」：全球边缘节点 + GSLB DNS 智能解析 → 用户访问最近节点。三层链路：浏览器 → CDN 边缘 → Nginx → 源站。
>
> 抓住三点：GSLB 调度、缓存命中/回源机制（X-Cache: HIT/MISS）、HTTP 缓存控制头（Cache-Control/ETag）。实际经验：静态资源 hash 指纹解决缓存更新。
