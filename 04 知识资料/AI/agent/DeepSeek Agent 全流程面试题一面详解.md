# DeepSeek Agent 全流程面试题逐题详解（一面）

> 原文：[微信文章](https://mp.weixin.qq.com/s/-X8oqp0jOVcE70FNq2YcaA) · 2026-07-01

## 一句话总结

DeepSeek Agent 一面涵盖 12 道技术深度题——从完整链路（请求→意图→Skill→规划→记忆→Prompt→执行→校验→输出→沉淀）到分层 Skill 体系、长短期记忆、动态 Prompt 组装、上下文 Token 管控，全链路覆盖。

---

## 一、Agent 完整运转链路（10 步）

```
1. 请求接入与预处理
   → 清洗、脱敏、标准化，拉取对话上下文和用户画像
2. 意图识别 & 任务分类
   → 判定：闲聊 / 简单问答 / 复杂工具任务 / 代码开发
3. Skill 匹配与调用决策
   → 检索 Skill 库，匹配已有模板 or 从头规划
4. 任务拆解与规划
   → 拆分子任务，标注串行/并行、执行方式（推理/工具）
5. 记忆召回
   → 短期对话记忆 + 长期静态/动态记忆，筛选强相关内容
6. 动态 Prompt 组装
   → 拼接：静态人设 + 召回记忆 + 历史对话 + 子任务指令 + 工具格式
7. 执行环节
   → 纯推理：模型直接生成 / 工具类：生成参数→调用工具→拿结果
8. 结果校验与迭代
   → 子任务达标？失败→重试/回滚/重规划；成功→推进下一个
9. 结果整合输出
   → 多轮子任务结果整理为自然语言回复
10. 记忆沉淀
    → 更新短期记忆，长价值信息写长期记忆，符合条件的流程沉淀为 Skill
```

---

## 二、意图理解与任务拆解

### 意图理解两步走

1. **粗粒度分类**：Prompt + 分类微调模型 → 闲聊/信息查询/工具操作/工程开发/多步骤规划
2. **细粒度槽位抽取**：目标对象、时间范围、操作权限、格式要求、验收标准、依赖资源

### 任务拆解两种范式

| 范式 | 适用 | 做法 |
|------|------|------|
| **启发式拆解** | 简单任务 | CoT 按逻辑拆分：查代码→改参数→验证→提交 MR |
| **结构化拆解** | 复杂任务 | 分层规划：大阶段→原子子任务，标注依赖/前置条件/兜底方案 |

---

## 三、执行顺序与模型/工具调用判定

### 执行顺序

- 基于依赖拓扑排序（有前置依赖的排前面）
- 无依赖子任务**并行执行**
- 高风险、高前置性任务优先

### 调用判定规则

| 调用大模型 | 调用工具 |
|-----------|---------|
| 主观推理、逻辑创作、自然语言润色 | 访问外部真实环境、确定性操作 |
| 复杂方案设计、模糊歧义决策 | 海量数据检索、代码运行、API |

> 混合模式是常态：模型做决策 + 工具拿结果，循环推进。

---

## 四、Skill 匹配策略

```
粗召回（向量检索TopN + 关键词匹配）
  → 规则过滤（权限、业务场景、生效范围）
    → 精排打分（大模型评匹配度、历史成功率、成本）
      → 兜底：匹配度低于阈值 → 不走已有Skill，从头规划
```

---

## 五、Skill 分层体系

| 层级 | 定位 | 示例 | 职责边界 |
|------|------|------|---------|
| **业务顶层** | 完整业务场景 | 电商大促全流程配置 | 理解目标、编排中层、把控验收 |
| **领域中层** | 垂直领域通用能力 | 后端接口 CRUD 代码生成 | 完成领域闭环、编排原子Skill |
| **原子底层** | 最小不可拆分 | 读 Git 文件、执行 Shell、向量检索 | 只做单一确定动作，不做业务判断 |

**分层原因**：上层业务迭代不碰底层，底层优化全链路受益，故障可快速定位到对应层级。

---

## 六、Skill 沉淀机制

| 模式 | 触发条件 | 说明 |
|------|---------|------|
| **自动沉淀** | 多次重复执行 + 成功率稳定 + 流程固定 | 抓取规划+执行流程，抽象触发条件+参数模板，生成候选Skill→人工审核 |
| **人工构造** | 新业务、无历史数据 | 专家定制顶层/中层Skill，固化最佳实践 |
| **审核上线** | 候选通过规则校验+风险评审 | 入库后持续监控，效果衰减→迭代或下线 |

---

## 七、长短期记忆设计

| 维度 | 短期记忆 | 长期记忆 |
|------|---------|---------|
| **存储位置** | 内存/高速缓存 | 持久化存储 |
| **保存内容** | 当前对话最近 N 轮（提问/回复/工具入参&结果/中间推理） | 用户偏好、历史需求、行业知识、经典方案、项目上下文 |
| **生命周期** | 对话结束归档筛选 | 跨会话持久化 |

---

## 八、静态 vs 动态长期记忆

| 维度 | 静态长期记忆 | 动态长期记忆 |
|------|------------|------------|
| **变更频率** | 低频 | 高频迭代 |
| **内容** | 企业技术规范、行业标准、固定架构、永久偏好 | 近期项目细节、临时习惯、历史踩坑 |
| **更新** | 严格审核 | 增量写入、定期整理 |
| **召回** | 稳定策略 | 结合时效性排序 |

**拆分价值**：针对性做存储/索引/权限优化，避免动态内容干扰静态知识的召回精度。

---

## 九、长期记忆防膨胀机制

| 层 | 做法 |
|----|------|
| **写入准入** | 只存有长期复用价值的，闲聊/临时参数/失败重试日志直接过滤 |
| **信息压缩** | 大模型先提炼核心结论做摘要，不存原始对话原文 |
| **分级过期** | 动态记忆设 TTL，按访问热度衰减，长期不召回→归档/清理 |
| **分库分表** | 按用户/业务线分片，冷热分离（热向量库/冷归档） |

---

## 十、记忆召回与防上下文污染

### 召回逻辑

Query 重构（多维度：业务领域/技术栈/时间/操作对象）→ 静态+动态记忆库多维度向量+关键词检索 → 初筛候选集。

### 防污染五层控制

1. **召回数量硬上限**：单轮 TopK 限制
2. **相关性精排**：大模型逐条打分，低于阈值丢弃
3. **时效性权重**：动态记忆优先近期，过时降分
4. **冗余去重**：核心信息重复的条目合并
5. **上下文配额**：记忆分配固定 token 配额，超配额精简摘要

---

## 十一、动态 vs 静态 Prompt

| 维度 | 静态 Prompt | 动态 Prompt |
|------|-----------|-----------|
| **变化** | 固定不变 | 每轮实时生成 |
| **内容** | Agent 人设、通用约束、输出格式、安全红线 | 当前输入、历史对话、召回记忆、工具列表、子任务指令 |
| **携带** | 每次请求完整携带 | 每轮不同 |

**组装流程**：静态底座 → 长期记忆（高相关靠前）→ 裁剪短期对话 → 子任务指令+工具清单 → 用户当前输入。全程 token 校验，超出就压缩最早的/最不重要的。

---

## 十二、上下文 Token 管控与压缩

| 阶段 | 做法 |
|------|------|
| **事前配额** | 静态 Prompt 基础配额 + 记忆/历史/输入/工具分块分配 |
| **事中监控** | 每拼接一段计算 token，临近阈值启动压缩 |
| **轻度压缩** | 历史对话摘要，去语气词/重复，保留关键决策和结果 |
| **中度压缩** | 合并多轮工具调用中间日志，只留入参和最终结论 |
| **重度压缩** | 舍弃最早、最不相关的历史轮次，只保留强相关 |
