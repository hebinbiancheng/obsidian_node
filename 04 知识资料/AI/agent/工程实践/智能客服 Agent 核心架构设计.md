# 智能客服 Agent 核心架构设计

> 原文：[微信文章](https://mp.weixin.qq.com/s/6awQQ-863A1d6xTJEwB-tA) · 2026-07-20
> 原始资料：`^[raw/articles/wechat-customer-service-agent-2026.html]`

## 一句话总结

智能客服 Agent = 接入层（统一消息 + 安全过滤）+ 理解层（意图/实体/情感）+ **处理层（四路径分派）** + 输出层（流式回复）。转人工机制是面试官最爱追问的点。

---

## 四层架构

### 第一层：接入层

- 多渠道接入：网页、APP、微信、电话 → 统一消息格式
- 敏感词过滤
- 请求限流（防恶意刷接口）

### 第二层：理解层

| 能力 | 做什么 |
|------|--------|
| **意图识别** | 判断用户想做什么（查订单、退货、投诉） |
| **实体抽取** | 提取关键信息（订单号、商品名、时间） |
| **情感分析** | 识别用户情绪，负面情绪及时预警 |

### 第三层：处理层（核心分派）

| 用户需求 | 处理方式 |
|---------|---------|
| 知识问答 | RAG 检索知识库回答 |
| 订单查询 | Function Calling 对接订单系统 |
| 投诉处理 | 创建工单 + 安抚回复 |
| 复杂/敏感问题 | **直接转人工** |

### 第四层：输出层

- 生成自然语言回复
- 确认关键信息，引导下一步
- **流式输出**：用户不用等全部生成完

---

## 转人工触发条件

| 条件 | 说明 |
|------|------|
| 用户明确要求 | "我要找人工" |
| 情绪激烈 | 连续多次负面情绪 |
| 超出知识库范围 | 无可匹配答案 |
| 连续 N 次未解决 | 同一问题反复失败 |

---

## 真实落地场景

| 场景 | 流程 |
|------|------|
| 电商客服 | "订单什么时候到" → 意图识别 → 调用订单 API → 返回物流 |
| 银行客服 | "信用卡账单" → 意图识别 → 调用账单 API → 返回应还金额 |
| 运营商客服 | "网速慢" → 意图识别 → 调用网络诊断 → 优化建议或派单 |

---

## 效果评估四指标

| 指标 | 含义 |
|------|------|
| 问题解决率 | 无需人工介入的比例 |
| 用户满意度评分 | 对话结束后的评分 |
| 平均对话轮数 | 解决问题平均要几轮 |
| 首次响应时间 | 用户发消息到第一句回复的延迟 |
