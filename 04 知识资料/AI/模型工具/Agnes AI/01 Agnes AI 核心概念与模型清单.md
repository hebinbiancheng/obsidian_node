# Agnes AI 核心概念与模型清单

## 1. Agnes AI 是什么

根据原文，Agnes AI 开放了核心多模态模型 API，作者关注它的原因是：在 Claude Code + Opus 做 Agent 项目时，长任务会消耗大量 token，而 Agnes 的免费 API 可以作为低成本执行层接入工作流。

## 2. 原文提到的模型

| 类型 | 模型 | 用途 |
|---|---|---|
| 文本模型 | `Agnes-2.0-Flash` / `agnes-2.0-flash` | 代码分析、文档生成、网页生成等文本任务 |
| 图片模型 | `Agnes-Image-2.1-Flash` | 架构图、流程图、创意图片生成 |
| 视频模型 | `Agnes-Video-2.0` | 短视频、动画、漫剧风格视频生成 |

## 3. 原文给出的首周数据

原文提到 Agnes 公布过一组首周数据：

- 文本模型 `Agnes-2.0-Flash` 首周消耗超过 1 万亿 token；
- 图片模型 `Agnes-Image-2.1-Flash` 生成 200 多万张图；
- 视频模型 `Agnes-Video-2.0` 累计生成 200 多万秒视频。

> 说明：这些数据来自原文转述，本文未额外验证官方实时统计。

## 4. 文章核心判断

作者的判断是：

- Agnes 不一定能替代 Claude Opus 做最复杂的规划和推理；
- 但免费、多模态、可接入 Claude Code 的特性，使它很适合承担大量试错型任务；
- 文本、图像、视频三个方向都可以进入开发者工作流；
- 更合理的组合是：高能力模型负责规划，Agnes 负责执行和多版本生成。

## 5. 适合使用的任务

- 代码库初步扫描；
- 项目结构文档生成；
- 核心模块说明；
- 静态网页 demo；
- 架构图 / 流程图生成；
- 封面图、插画、国潮图等视觉素材；
- 短视频脚本到视频的快速试错。

## 6. 不建议直接重度依赖的任务

- 直接修改核心生产代码；
- 高风险自动化批量改动；
- 一次性要求高稳定输出的长任务；
- 对事实准确性、版权、安全边界要求很高的内容；
- 不经人工筛选就批量投产的图片和视频素材。
