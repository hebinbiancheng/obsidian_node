# Agnes AI 原文解析要点

## 原文主题

原文围绕“如何将 Agnes AI 免费 API 接入 Claude Code，并评估文本、图像、视频模型是否可用”展开。

## 原文结构

| 部分 | 内容 |
|---|---|
| 背景 | Claude Code + Opus 做 Agent 项目 token 消耗大，引出免费 API 需求 |
| 模型清单 | 文本、图片、视频三类 Agnes 模型 |
| Claude Code 接入 | 通过 CC Switch 将 Claude Code 请求路由到 Agnes API |
| 文本模型实测 | 项目源码解析、静态网页生成 |
| 图像模型实测 | 架构图、流程图、offer 场景图、金榜题名图 |
| 视频模型实测 | 龙舟比赛、卡通小狐狸学编程 |
| 使用建议 | Opus 规划，Agnes 执行；小任务试水，多版本试错 |

## 关键事实摘录

- API 地址：`https://apihub.agnes-ai.com/v1`
- API Platform：<https://platform.agnes-ai.com/>
- CC Switch：<https://github.com/farion1231/cc-switch/releases>
- 文本模型：`agnes-2.0-flash`
- 图像模型：`Agnes-Image-2.1-Flash`
- 视频模型：`Agnes-Video-2.0`

## 关键观点

1. 免费 token 的价值在于降低试错成本；
2. Agnes 适合作为 Claude Code 的执行层；
3. 文本模型可以做项目结构分析和静态页面生成；
4. 图像模型可以生成架构图、流程图和创意图；
5. 视频模型适合短视频候选生成；
6. 不建议把免费模型直接用于高风险生产任务；
7. 更合理的模型分工是：强模型做规划，免费模型做执行。
