# Agnes 文本模型实测与适用场景

## 1. 原文测试目标

作者用 `agnes-2.0-flash` 测了两类任务：

1. 读项目代码并生成结构文档；
2. 从零生成一个可运行的静态网页 demo。

这两个任务分别测试：

- 代码阅读、文件扫描、模块归纳能力；
- 需求理解、前端生成、完整交付能力。

## 2. 任务一：源码解析

### 测试任务

作者将 MewCode Agent 项目源码交给模型，要求产出两份文档：

1. 项目结构概览：主要目录、目录职责、入口文件；
2. 核心模块说明：挑出 3 个关键模块，说明职责、接口、调用关系。

要求包括：

- 不罗列所有文件；
- 只挑主要业务逻辑；
- 模块说明要带文件路径；
- TODO / FIXME / 临时实现单独标注。

### 原文观察

模型会启动 sub agent 扫描源码，再按业务主线归纳。输出结果包括：

- 项目结构文档；
- 核心模块说明；
- 文件路径；
- 模块调用关系；
- TODO / FIXME 标注。

### 可复用 Prompt 模板

```text
请帮我分析这个项目的整体结构，重点产出两份文档：

1. 项目结构概览：列出主要目录、各目录职责、入口文件；
2. 核心模块说明：挑出 3-5 个最关键模块，说明每个模块的职责、对外接口、与其他模块的调用关系。

要求：
- 不要罗列所有文件，只挑承担主要业务逻辑的文件；
- 模块说明要带具体文件路径；
- 如果发现 TODO、FIXME 或明显临时实现，单独列一节；
- 最后给出后续阅读顺序建议。
```

## 3. 任务二：静态网页生成

### 测试任务

作者要求生成一个世界杯论坛单页 demo：

- 顶部导航；
- 左侧帖子列表；
- 右侧当日赛程卡片；
- 至少 5 条帖子；
- 至少 3 场比赛；
- 单 HTML 文件；
- CSS 写在 `<style>` 中；
- 世界杯主题配色。

### 原文观察

模型生成的页面满足主要要求：

- 顶部导航存在；
- 左侧帖子列表完整；
- 右侧赛程卡片完整；
- 头像、用户名、回复数等细节齐全；
- 配色符合世界杯主题。

### 可复用 Prompt 模板

```text
请用纯前端做一个 [主题] 的单页 demo，文件名 [filename].html。

页面要求：
1. 顶部导航，左侧是站点 logo，右侧是登录/注册按钮；
2. 主体分两栏，左侧是内容列表，右侧是信息卡片；
3. 列表至少 5 条，每条包含头像、用户名、标题、时间、互动数据；
4. 信息卡片至少 3 条；
5. 全部静态写在 HTML 里，不要外部依赖，样式写在 <style> 标签里；
6. 配色使用 [主题色]，并有适当点缀色。

要求：
- 只生成一个 HTML 文件；
- 打开即可运行；
- 末尾注释说明哪些地方建议接后端。
```

## 4. 上下文能力

原文提到：

- 当前文本模型支持 256K 上下文；
- 官方称本周灰度升级到 1M 上下文。

> 说明：这是原文在 2026-06-13 的描述，具体上线状态需要以 Agnes 官方文档为准。

## 5. 适用场景总结

适合：

- 项目初读；
- 结构文档生成；
- README 初稿；
- 静态 demo；
- 低风险代码解释；
- 多版本草稿生成。

谨慎使用：

- 直接修改核心业务代码；
- 涉及生产数据或安全逻辑；
- 长链路自动化任务；
- 需要强事实准确性的输出。
