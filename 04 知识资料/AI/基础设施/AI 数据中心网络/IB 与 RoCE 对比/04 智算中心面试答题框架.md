# 智算中心面试答题框架

## 1. 面试题原型

> 从协议栈、流控机制、无损网络实现方式、典型时延/吞吐、大规模运维复杂度五方面对比 IB 和 RoCE，并说明各自适用场景。

这道题考察的不是背概念，而是你是否理解智算网络的底层逻辑：RDMA 为什么需要无损网络、为什么长尾延迟会影响训练效率、为什么 RoCE 的性价比背后伴随复杂调优。

## 2. 30 秒核心回答

IB 和 RoCE 都服务于 RDMA 高性能通信，但实现路径不同。IB 是专有网络，原生 RDMA、硬件 Credit 流控、天然无损，延迟低且长尾稳定，适合超大规模训练和 HPC，但成本高、生态封闭。RoCEv2 基于以太网和 UDP/IP，通过 PFC + ECN + DCQCN 构造准无损网络，成本低、生态开放、可与现有数据中心网络融合，但对配置、监控和拥塞调优要求高。选型上，极致训练性能和确定性选 IB，成本敏感、中大规模、以太网团队强的场景选 RoCEv2。

## 3. 五维对比模板

### 维度一：协议栈

- IB：独立协议栈，由 IBTA 体系定义，传输层逻辑由 HCA 硬件处理；
- RoCEv2：RDMA over UDP/IP Ethernet，可跨三层网络，更容易融入现有数据中心。

### 维度二：流控机制

- IB：Credit-based Flow Control，发包前确认接收端 buffer；
- RoCEv2：PFC 负责二层暂停，ECN 标记拥塞，DCQCN 端到端降速。

### 维度三：无损实现

- IB：协议内生天然无损；
- RoCEv2：工程配置形成准无损，依赖队列、水位线、QoS、ECN 阈值。

### 维度四：时延与吞吐

- IB：低延迟、长尾稳定，带宽利用率高；
- RoCEv2：端口速率可相同，但拥塞时长尾更容易恶化，ECMP 哈希和调优质量影响整体吞吐。

### 维度五：大规模运维

- IB：依赖 SM，生态封闭，专门人才较少；
- RoCEv2：基于 BGP/SDN 扩展性强，但 PFC 死锁、拥塞扩散、INT 监控和自动化排障难度高。

## 4. 对比矩阵

| 维度 | IB | RoCEv2 |
|---|---|---|
| 协议 | 独立 IB 协议栈 | Ethernet + IP + UDP |
| RDMA | 原生支持 | 在以太网上承载 RDMA |
| 无损 | Credit 硬件流控 | PFC + ECN + DCQCN |
| 延迟 | 极低且稳定 | 较低但长尾更敏感 |
| 成本 | 高 | 相对低 |
| 生态 | 封闭、NVIDIA 主导 | 开放、多供应商 |
| 运维 | SM、专有工具 | BGP/SDN、遥测、自动化 |
| 场景 | 万卡训练、HPC | 中大规模训练、推理、存储、成本敏感场景 |

## 5. 加分回答

面试最后可以补一句趋势判断：

- UEC（Ultra Ethernet Consortium）正在推动面向 AI/HPC 的增强以太网；
- 目标是把多路径、链路层重试、Credit 类流控等能力引入以太网；
- UALink、UEC、NVIDIA Spectrum-X 等方向都说明：行业正在尝试弥合 IB 的性能确定性与以太网开放生态之间的差距。
