# 协议栈、流控与无损网络机制

## 1. 协议栈差异

### InfiniBand

IB 使用独立协议栈，不依赖传统 TCP/IP。传输逻辑大量由 HCA 网卡硬件处理，因此可以降低 CPU 参与度和协议栈开销。

特点：

- 独立物理层、链路层、网络层和传输层；
- HCA 负责数据搬运和协议处理；
- 与 HPC / MPI / NCCL 等高性能计算场景深度绑定；
- 专有生态更强。

### RoCEv2

RoCEv2 把 RDMA 报文封装在 UDP/IP 中，默认目的端口常见为 4791。它的优势是可以融入标准以太网与三层路由体系。

特点：

- 兼容 Ethernet + IP 网络；
- 可跨 L3 路由；
- 适合与 BGP、ECMP、SDN、SONiC 等体系结合；
- 但必须额外构造无损能力。

## 2. Credit 流控 vs PFC 暂停帧

### IB：Credit Based Flow Control

IB 的核心是 Credit 流控：发送端发包前先确认接收端还有 buffer 空间。如果没有 Credit，就不会发送。

这类机制的优势是：

- 从源头避免 buffer 溢出；
- 硬件级逐跳控制；
- 延迟稳定；
- 更适合大规模同步训练。

### RoCEv2：PFC + ECN + DCQCN

以太网天然是 Best Effort。为了让 RoCE 跑 RDMA，需要组合机制：

- **PFC**：当特定优先级队列水位过高时发送 PAUSE 帧，暂停上游发送；
- **ECN**：在 IP 层标记拥塞，而不是等到丢包；
- **DCQCN**：通过端到端拥塞控制让发送端降速。

这套机制可以构造准无损网络，但难点是：

- PFC 水位线配置复杂；
- headroom 计算随链路速率上升而更敏感；
- PFC 可能造成反压扩散、死锁或拥塞树；
- ECN/DCQCN 参数不合理会导致吞吐和长尾抖动。

## 3. 无损网络实现方式

| 维度 | InfiniBand | RoCEv2 |
|---|---|---|
| 无损属性 | 协议内生 | 人工构造 |
| 核心机制 | Credit 流控 | PFC + ECN + DCQCN |
| 控制位置 | 硬件逐跳 | 二层反压 + 三层标记 + 端到端降速 |
| 风险 | SM、生态绑定 | PFC 风暴、死锁、阈值配置错误 |
| 运维关注 | 拓扑、SM、链路健康 | 队列、水位、ECN、INT、拥塞热点 |

## 4. 为什么“丢包”在 AI 训练里不是小问题

分布式训练的通信往往存在同步等待。一次 All-Reduce 中，如果某个节点因丢包或重传多等几毫秒，其他 GPU 也可能被迫等待。单次延迟看似很小，但在十万轮迭代、几千块 GPU 的规模上，会放大成显著的 GPU 利用率损失。
