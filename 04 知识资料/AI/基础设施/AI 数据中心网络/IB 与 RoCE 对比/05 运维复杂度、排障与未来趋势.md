# 运维复杂度、排障与未来趋势

## 1. IB 的运维特点

IB 的优势是更接近“开箱即用”的高性能训练网络，但并不代表没有运维复杂度。

关键组件：

- **HCA**：Host Channel Adapter，负责 RDMA 和协议处理；
- **Subnet Manager, SM**：负责拓扑发现、LID 分配、路由下发；
- **IB 交换机与线缆**：通常需要遵循同一生态兼容性。

主要风险：

- SM 在极大规模下的性能和主备切换；
- 供应商绑定；
- 专有工具链和人才稀缺；
- 硬件替换和扩容灵活性较低。

## 2. RoCEv2 的运维特点

RoCE 的优势是开放和成本，但对网络工程能力要求更高。

核心运维对象：

- PFC 优先级与队列；
- XON/XOFF 水位线；
- ECN 标记阈值；
- DCQCN 参数；
- ECMP 哈希与多路径均衡；
- INT / Telemetry 监控；
- 丢包、重传、PFC 暂停帧、队列水位、链路拥塞的可观测性。

典型问题：

- PFC 风暴；
- PFC 死锁；
- headroom 不足导致丢包；
- ECN 过早或过晚触发；
- ECMP 哈希不均造成热点；
- 长尾延迟不稳定导致训练效率下降。

## 3. RoCE 排障检查清单

- [ ] RDMA 流量是否进入正确优先级队列？
- [ ] PFC 是否只对必要优先级启用，避免全局暂停？
- [ ] 每个端口 headroom 是否按链路速率和线缆距离计算？
- [ ] ECN 阈值是否过高导致拥塞发现太晚？
- [ ] DCQCN 是否有效降低发送速率？
- [ ] ECMP 是否存在 elephant flow 哈希冲突？
- [ ] 是否有 PFC pause storm 或 pause duration 异常？
- [ ] 是否有 NIC / switch 侧 RDMA retransmit、CNP、ECN mark 指标？
- [ ] 是否部署 INT 或等价遥测来定位拥塞路径？

## 4. 未来趋势：UEC 与增强以太网

原文一提到 2025 年 6 月 UEC 1.0 规范发布。其方向是让以太网具备更适合 AI/HPC 的能力，例如：

- 多路径喷发与接收端重排；
- 链路层重试；
- 类 Credit 的流量控制；
- 更强的硬件拥塞控制。

如果 UEC 产品在 2025-2026 年持续落地，以太网与 IB 的差距可能继续缩小。

## 5. NVIDIA 的双线策略

原文一也指出 NVIDIA 并不是只押 IB：

- NVLink：机柜内 Scale-Up；
- Quantum-X：IB Scale-Out 训练；
- Spectrum-X：以太网，用于推理和通用数据中心连接；
- Spectrum-X Photonics / CPO：降低端口功耗，提升大规模以太网连接能力。

这说明：未来 AI 网络很可能不是单一技术胜出，而是 Scale-Up、训练 Scale-Out、推理网络、存储网络分层组合。

## 6. 实战建议

- 如果团队没有强网络调优能力，不要低估 RoCE 的隐性运维成本；
- 如果预算紧张，也不要只看 IB 的性能优势，要计算 TCO 和可购买 GPU 的机会成本；
- 对新建集群，建议把网络选型与训练框架、NCCL 参数、监控体系、故障演练一起设计；
- 对 RoCE 集群，必须建设指标体系和自动化排障，而不是只配置一次 PFC/ECN 就结束。
