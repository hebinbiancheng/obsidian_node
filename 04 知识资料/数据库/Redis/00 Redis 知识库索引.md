## 一句话总结

Redis 是一种基于内存的高性能键值数据库，支持丰富的数据结构（String、Hash、List、Set、ZSet 等），具备持久化、集群、事务、发布订阅等能力，广泛应用于缓存、分布式锁、消息队列、排行榜等场景。

## 核心地图

```text
Redis
├── 数据结构层
│   ├── String → SDS
│   ├── Hash → ziplist / hashtable
│   ├── List → quicklist
│   ├── Set → intset / hashtable
│   └── ZSet → ziplist / skiplist + dict
├── 线程模型
│   ├── 单线程 IO + 命令处理
│   ├── IO 多路复用 (epoll)
│   └── Redis 6.0+ 多线程 IO
├── 持久化
│   ├── RDB：快照，二进制，恢复快
│   ├── AOF：命令日志，实时性好
│   └── 混合持久化：RDB + AOF
├── 高可用
│   ├── 主从复制：读写分离
│   ├── 哨兵：自动故障转移
│   └── Cluster：数据分片
├── 内存管理
│   ├── 过期删除：惰性 + 定期
│   └── 内存淘汰：8 种策略
└── 缓存设计
    ├── 三大问题：穿透、击穿、雪崩
    └── 更新策略：Cache Aside
```
