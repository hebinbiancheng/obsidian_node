# 01 Go 与其他语言对比

> 来源：[[04 知识资料/编程语言/Go/source/Go 原始 XMind 解析.md]]
  - 1.go与其他语言对比
    - 1、Go语言和c++有什么区别? 也与其他语言相比
      - Go语言和C++在设计理念、用途和特性上有显著的区别：
      - 1. 设计哲学和简洁性
        - Go：Go语言提供了垃圾回收、简洁的并发模型（通过goroutines和channels）、以及只有一个编程方式的标准库。Go的语法简洁，去除了很多其他语言中常见的复杂性（如类和继承）。
        - C++：C++是一种复杂的语言，提供了丰富的特性，如类、模板、多重继承、运算符重载等。
      - 2. 内存管理
        - Go：自动内存管理，使用垃圾回收机制来管理内存。这减少了内存泄漏的风险，但可能会影响性能（如垃圾回收暂停时间）。
        - C++：手动内存管理，程序员需要负责分配和释放内存。这允许更精细的控制，但也增加了内存泄漏和其他内存错误的风险。
      - 3. 并发编程
        - Go：并发是Go的一大特色，通过goroutines（轻量级线程）和channels来实现。Go的并发模型被设计为"不要通过共享内存来通信；而是通过通信来共享内存"。
        - C++：传统上依赖于线程库（如POSIX线程）和在C++11及以后版本中引入的线程支持库。C++的并发编程通常更复杂，需要处理同步和数据竞争等问题。
      - 4. 标准库和第三方库
        - Go：有一个广泛的标准库，几乎涵盖了所有基本的程序需求，如网络编程、加密、数据处理等。Go的标准库被设计为足够用而不过度膨胀。
        - C++：标准库提供了基本数据结构、数学运算、文件处理等，但对于网络编程、数据库操作等则需要依赖第三方库。
      - 5. 用途
        - Go：由于其简洁和高效的并发模型，Go特别适合构建网络服务和分布式系统。例如，Docker和Kubernetes
        - C++：由于其性能优势和系统级控制，C++广泛用于系统软件、游戏开发、实时系统、高性能计算等领域。
      - 6. 编译和运行
        - Go：Go有一个非常快速的编译器，支持跨平台编译，并且生成的是静态链接的二进制文件，易于部署。
        - C++：C++编译通常比Go慢，尤其是涉及大型项目和复杂模板时。生成的可执行文件可能需要依赖特定的库。
      - 总的来说，选择Go语言还是C++取决于项目的具体需求、团队的熟悉程度以及期望的性能和复杂性。两种语言各有优势和适用场景
    - 2、Go 是面向对象的语言吗？
      - 是的，也不是。 原因解释
        - 1、Go 有类型和方法，并且允许面向对象的编程风格，但没有类型层次。
        - 2、Go 中的 "接口 "概念提供了一种不同的方法，而且在某些方面更加通用。还可以将类型嵌入到其他类型中，以提供类似的东西，但不等同于子类。
        - 3、Go 中的方法比 C++ 或 Java 中的方法更通用：它们可以为任何类型的数据定义，甚至是内置类型，如普通的、"未装箱的 "整数。它们并不局限于结构（类）。
        - 4、Go 由于缺乏类型层次，Go 中的 "对象 "比 C++ 或 Java 等语言更轻巧。
    - 3、Go 实现面向对象编程
      - 封装
        - 面向对象中的 “封装” 指的是可以隐藏对象的内部属性和实现细节，仅对外提供公开接口调用，这样子用户就不需要关注你内部是怎么实现的。
        - 在 Go 语言中的属性访问权限，通过首字母大小写来控制：
          - 首字母大写，代表是公共的、可被外部访问的。
          - 首字母小写，代表是私有的，不可以被外部访问。
        - 例子
      - 继承
        - 面向对象中的 “继承” 指的是子类继承父类的特征和行为，使得子类对象（实例）具有父类的实例域和方法，或子类从父类继承方法，使得子类具有父类相同的行为。
        - Go 语言没有类似 extends 关键字的这种继承的方式,采取的是组合的方式
          - 组合了子结构的方法和属性
          - 例子
      - 多态
        - 多态 面向对象中的 “多态” 指的同一个行为具有多种不同表现形式或形态的能力，具体是指一个类实例（对象）的相同方法在不同情形有不同表现形式。 多态是通过接口来实现
          - 例子
