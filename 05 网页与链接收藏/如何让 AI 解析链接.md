# 如何让 AI 解析链接

## 推荐安装

### 必装

- `Claudian`：让 Codex/Claude 读取 vault、整理笔记、写入分析结果。
- `Dataview`：让首页仪表盘自动展示最近更新、待办、待读链接。
- `Obsidian Web Clipper`：把浏览器网页保存成 Markdown 笔记。

### 建议安装

- `QuickAdd`：快速新建链接记录、文章分析任务。
- `Templater`：创建带日期、来源、状态的模板。
- `Buttons`：在首页放真正可点击执行的按钮。
- `Advanced URI`：让按钮打开命令、文件、搜索或 QuickAdd 动作。
- `Tasks`：更强的任务筛选、到期时间、完成状态管理。

## 普通网页

1. 把网址放入 [[待读链接/链接收集箱|链接收集箱]]。
2. 对 Claudian/Codex 说：

```text
请读取链接收集箱中未完成的普通网页链接，抓取或解析正文，
按「06 模板/知识库条目模板.md」生成知识库笔记，
保存到「04 知识资料/对应专题」，并更新专题索引和知识库总索引。
原始链接和处理记录可以保留到「05 网页与链接收藏/已解析内容」。
```

如果 AI 无法联网或网页反爬，使用浏览器 Web Clipper 保存网页正文，再让 AI 分析本地 Markdown。

## 微信文章

微信文章经常有登录、反爬、动态加载限制。推荐流程：

1. 在浏览器打开微信文章。
2. 用 Web Clipper 保存正文到 `05 网页与链接收藏/微信文章`。
3. 如果保存失败，手动复制正文到新笔记。
4. 让 AI 分析本地笔记，而不是直接分析链接。

推荐提示词：

```text
请分析「05 网页与链接收藏/微信文章」里最新的文章，
提取核心观点、概念、可复用知识点和行动建议，
按「06 模板/知识库条目模板.md」保存到「04 知识资料/对应专题」，
并更新专题索引和知识库总索引。
```

## 抖音链接

抖音链接通常不能稳定直接解析视频内容。推荐流程：

1. 保存分享链接到 [[待读链接/链接收集箱|链接收集箱]]。
2. 手动补充标题、简介、评论中有价值的信息。
3. 如果需要分析视频内容，先提供字幕、转写文本或截图。
4. AI 根据你提供的文本进行总结和归档。

推荐提示词：

```text
下面是一个抖音视频的标题、简介和字幕。请总结核心内容，
判断它适合归档到「04 知识资料」哪个专题，
并按「06 模板/知识库条目模板.md」生成一篇 Obsidian 知识库笔记。
```

## 归档规则

- 临时未读：`05 网页与链接收藏/待读链接`
- 原始链接和剪藏正文：`05 网页与链接收藏/已解析内容`
- 长期知识：默认保存到 `04 知识资料/对应专题`
- 项目相关：如果主要是项目执行资料，保存到 `02 项目`，同时把可复用知识沉淀到 `04 知识资料`
- 新专题：如果没有合适目录，可在 `04 知识资料` 下创建通俗易懂的新目录

## 知识库输出标准

每篇解析后的知识库笔记必须包含：

- YAML：`type: knowledge`、`status: evergreen`、`source`、`source_type`、`created`、`updated`、`tags`、`aliases`
- 正文：一句话结论、核心概念、详细笔记、常见问题、可复用知识点、原子笔记拆分建议、关联笔记、原文信息、后续行动
- 双链：至少链接到 [[04 知识资料/知识库总索引|知识库总索引]] 和一个专题索引
- 标签：至少包含 `知识库` 和 1-3 个专题标签
- 索引更新：把新笔记加入对应专题索引；必要时更新知识库总索引
