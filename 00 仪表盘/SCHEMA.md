# 知识库治理规范 (SCHEMA)

> 本文件定义 obsidian_note 知识库的结构规则、命名约定、标签分类法和质量标准。
> 所有知识库操作必须遵循本规范。新增标签前必须先在此登记。

## 领域范围

本知识库覆盖以下技术领域：

- **AI**：Agent、模型工具、基础设施、内容安全、Agent Skills
- **编程语言**：Go（为主）、其他语言
- **基础设施**：Kubernetes、运维
- **数据**：数据库（Redis、SQL）、消息队列（Kafka）
- **系统设计**：场景题、架构设计
- **面试**：技术复习、面经

## 命名约定

### 文件名
- 使用中文描述性名称，允许空格
- 知识库索引文件统一命名为 `00 <主题>知识库索引.md`
- 专题笔记按序号命名：`01 核心概念.md`、`02 关键结构.md` …
- 原子笔记使用主题名：`Go Slice.md`、`Redis 持久化.md`
- 禁止使用无意义的数字编号（如纯日期）

### 目录名
- 使用中文，层级不超过 5 层
- 专题目录名应通俗易懂
- 每个专题目录下必须有 `assets/` 子目录存放图片

## Frontmatter 规范

### 知识库条目（type: knowledge）

```yaml
---
type: knowledge
status: evergreen | draft | archived
source: "原始来源链接或引用"
source_type: 微信文章 | 知乎文章 | 网页 | 视频 | XMind | 其他
author: 作者名
published: YYYY-MM-DD
created: YYYY-MM-DD
updated: YYYY-MM-DD
tags:
  - 知识库
  - <领域标签>
  - <专题标签>
aliases:
  - 别名1
  - 别名2
confidence: high | medium | low
---
```

### 索引页（type: moc）

```yaml
---
type: moc
status: evergreen
tags:
  - 知识库
  - <领域标签>
---
```

### 必填字段
- `type`：knowledge / moc / project / daily / meta
- `status`：evergreen / draft / archived
- `created`：创建日期
- `tags`：至少包含 `知识库` + 1-3 个专题标签

### 推荐字段
- `confidence`：知识可信度（high=多源验证 / medium=单源但可靠 / low=观点或推测）
- `contested`：设为 true 表示内容存在争议
- `source`：原始来源

## 标签分类法

所有标签必须来自以下分类法。新增标签前必须先在此登记。

### 领域标签
`AI` `Go` `Kubernetes` `运维` `数据库` `消息队列` `系统设计` `面试`

### 专题标签

## 页面阈值

| 规则 | 阈值 |
|------|------|
| 创建新页面 | 内容可独立成篇（≥10行有意义内容） |
| 拆分页面 | 超过 200 行 |
| 合并页面 | 内容少于 20 行且主题高度重叠 |
| 归档页面 | 内容完全过时或已被替代 |
| 索引条目 | 每个专题索引至少包含 3 个条目 |

## 知识库条目质量标准

每个知识库条目必须包含：

- ✅ YAML frontmatter（含 type、status、tags、created）
- ✅ 一句话结论
- ✅ 核心概念
- ✅ 详细笔记（结构化章节）
- ✅ 常见问题（至少 2 个 Q&A）
- ✅ 可复用知识点
- ✅ 关联笔记（至少 2 个 [[双链]]）
- ✅ 原文信息

每个知识库索引必须包含：

- ✅ 一句话总结
- ✅ 知识库目录（双链列表）
- ✅ 核心地图（文本树形图）
- ✅ 适合沉淀的问题
- ✅ 关联笔记

## 图片与附件规则

- 图片统一存放在专题目录下的 `assets/` 子目录
- 命名格式：`<序号>-<语义描述>.png`，如 `01-architecture-overview.png`
- 使用 Obsidian 原生语法嵌入：`![[assets/图片名.png]]`
- 禁止使用外部图片链接（易失效）

## 维护节奏

| 频率 | 任务 |
|------|------|
| 每天 | 查看首页仪表盘和待做 |
| 每周 | 清理 `01 未整理笔记`，处理待读链接 |
| 每月 | 归档已完成项目，运行 lint 检查 |
| 每季 | 审查标签分类法，更新 SCHEMA.md |
