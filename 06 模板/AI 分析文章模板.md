# AI 分析文章模板

## 给 AI 的任务

请分析下面这些链接或文章内容，并按“知识库条目”的方式保存到 Obsidian：

- 链接：
- 原文位置：
- 我关心的问题：

## 输出要求

1. 判断资料类型：技术知识 / 项目资料 / 工具资源 / 观点文章 / 视频内容 / 其他。
2. 判断是否值得进入长期知识库；如果值得，默认保存到 `04 知识资料/对应专题`。
3. 如果已有合适专题目录，优先放入已有目录；如果没有，创建通俗易懂的新专题目录。
4. 使用 `06 模板/知识库条目模板.md` 的结构生成笔记。
5. 必须写 YAML frontmatter：`type: knowledge`、`status: evergreen`、`source`、`source_type`、`created`、`updated`、`tags`、`aliases`。
6. 必须包含：一句话结论、核心概念、详细笔记、常见问题、可复用知识点、原子笔记拆分建议、关联笔记、原文信息、后续行动。
7. 必须添加双链：链接到 `[[04 知识资料/知识库总索引|知识库总索引]]`，并链接到对应专题索引或相关笔记。
8. 必须添加标签：至少包含 `知识库` 和 1-3 个专题标签，例如 `AI`、`Kubernetes`、`运维`、`数据库`。
9. 如果内容很长，先生成一篇主笔记，再把 3-8 个独立主题拆成专题笔记，并互相双链。
10. 更新相关专题索引和 `04 知识资料/知识库总索引.md`。
11. 如果链接无法访问，说明原因，并告诉我需要复制正文、字幕、截图还是使用 Web Clipper。

## 推荐提示词

```text
请读取我指定的链接或正文，并按 Obsidian 知识库方式处理：

1. 提取标题、作者、发布时间、来源链接、核心内容。
2. 判断应该归档到 `04 知识资料` 下哪个专题目录；目录名要通俗易懂。
3. 按 `06 模板/知识库条目模板.md` 生成 Markdown 笔记。
4. 补充 YAML、标签、别名、双链、关联笔记和后续行动。
5. 如果内容包含多个独立主题，请拆成多篇专题笔记，并在主笔记中建立目录和双链。
6. 更新对应专题索引和 `04 知识资料/知识库总索引.md`。
7. 原始链接可以保留在 `05 网页与链接收藏/已解析内容`，但长期知识必须沉淀到 `04 知识资料`。
8. 如果链接无法访问，请说明原因，并告诉我需要提供正文、字幕、截图或 Web Clipper 保存结果。
```
